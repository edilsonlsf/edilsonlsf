📋 Notas do Protótipo — Pontos de Validação com o Usuário

TELA 01 · Login
Facilidade de acesso
Testar se o usuário prefere login social (Google/Apple) ao tradicional e-mail/senha. Reduz atrito inicial, especialmente para novos clientes.
TELA 02 · Home
Atalhos para ações mais usadas
Validar se o cliente de frotas e o cliente pessoa física navegam de forma distinta. O card "Alugar veículo" em destaque foi hipótese levantada durante imersão DT.
TELA 03 · Busca
Filtros relevantes para o contexto
Testar se os campos de data, local e categoria são suficientes ou se o usuário sente falta de filtros adicionais, como "pet-friendly" ou bagageiro.
TELA 04 · Resultados
Clareza das informações de comparação
Verificar se o usuário compreende os badges e badges de categoria. O destaque "Mais alugado" surgiu como elemento de prova social no workshop de DT.
TELA 05 · Confirmação
Transparência antes do pagamento
Ponto crítico: o usuário precisa ver resumo completo antes de pagar. Testar se há ansiedade sobre taxas ocultas — problema identificado na fase de empatia.
TELA 06 · Sucesso
Redução do tempo de retirada
O código de reserva visível e o comprovante digital são hipóteses para eliminar filas no balcão — principal dor mapeada no mapa de jornada do cliente.
